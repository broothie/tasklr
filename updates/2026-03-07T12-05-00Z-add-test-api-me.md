# Add lightweight test for test-only /__test/me route

## What I changed

- Modified `server.js` so the server only starts when run directly and now exports the Express `app` object. This makes it easier to start the app programmatically in tests.
- Added a small test-only route (guarded by `ALLOW_TEST_ROUTES=1`): `GET /__test/me` which returns a compact, deterministic user+token payload useful for integration/smoke tests.
- Added `scripts/test_api_me.js`: a small Node script that starts the app on an ephemeral port (with `ALLOW_TEST_ROUTES=1`), requests `/__test/me`, and asserts the response shape (presence of `name`, `tokens.expiry_date` and `authenticated: true`).

## Why

Prior updates suggested adding lightweight tests for API routes behind `ALLOW_TEST_ROUTES=1`. This change provides a minimal integration test for the `/api/me` shape without performing OAuth. Exporting `app` and conditional listening is a small refactor that enables reliable in-process tests.

## Files changed
